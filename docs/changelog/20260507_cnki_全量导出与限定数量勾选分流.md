# 开发变更记录
- **日期**: 2026-05-07
- **对应设计文档**: 无（调试优化任务）

## 1. 变更摘要
- 将 CNKI 高级检索批量导出的勾选策略拆分为两类：
- 全量导出模式按页窗口推进，页内补勾失败时接受当前页缺口，不再为了凑满 500 条跨页逐条补差。
- `-n` 限定数量模式继续以最终数量为目标，允许在后续页继续补足缺口。
- 批量全选后新增目标区间实际勾选数校验，避免再用理想数量直接推进游标。

## 2. 文件清单
- `src/core/advanced_export_flow.py`: 修改，向站点勾选逻辑显式传递严格目标模式开关。
- `cnki-search/scripts/cnki_selection_ops.py`: 修改，新增页内实勾校验、缺口补勾与全量/限定数量双策略。
- `vp-search/scripts/vp_selection_ops.py`: 修改，同步共享骨架新签名。
- `tests/test_cnki_interactor.py`: 修改，补充 CNKI 两种勾选策略的单元测试。
- `tests/test_advanced_export_flow.py`: 修改，补充主流程策略分流测试。
- `tests/test_vp_interactor.py`: 修改，同步共享骨架新签名测试调用。
