## 1. 变更摘要
优化万方高级检索与结果页操作链路的等待策略，避免 Playwright 默认导航等待把普通 SPA 点击拖到超时。

本次调整聚焦两点：
- 结果页等待改为只认下一步真实会消费的结果摘要元素。
- 下拉、过滤项、检索按钮、每页条数切换等交互统一使用动作超时，并关闭默认导航等待，再由代码显式等待目标状态。

补充修复：
- 年份 iView 下拉改为“点击后必须验值成功”，若值未变更会立即执行 JS 重试，并在严格模式下直接失败报错，避免“日志显示操作完成但实际仍是至今”。
- “中英文扩展”改为基于稳定文本节点读取 class，并在点击后显式等待 `active` 状态消失，避免点击成功后因 `.active` 选择器失效而误判超时。
- 导出页标签切换重构为“目标 tab 定位 + 目标状态等待”，只检查目标 tab 自身的 class 与目标导出控件是否就绪，不再固定读取第一个 tab 的 `active` 状态。

## 2. 文件清单
- `wanfang-search/scripts/wanfang_page_ops.py`: 修改结果页等待、目标页等待、每页条数等待逻辑
- `wanfang-search/scripts/wanfang_form_ops.py`: 修改检索按钮、iView 下拉、中英文扩展、每页条数切换点击策略，并补充下拉验值与重试逻辑
- `wanfang-search/scripts/wanfang_export_ops.py`: 重构导出页标签定位、激活态判定与就绪等待逻辑
- `wanfang-search/scripts/wanfang_navigation_ops.py`: 修改翻页和页码跳转等待逻辑
- `tests/test_wanfang_interactor.py`: 新增等待、点击策略、下拉严格校验、导出页目标状态测试
