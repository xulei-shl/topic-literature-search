## 1. 变更摘要
为 `wanfang-search / vp-search / cnki-search` 的截至年份逐年导出补充“年度结果数量校验后再总合并”的兜底逻辑：

- 全部年份首轮导出完成后，先校验每个年度目录中的汇总 `report.txt` 与 `merged.xlsx`
- 校验规则为：读取年度汇总报告中的 `总数`，并比对合并 Excel 的实际数据行数
- 对数量不一致的年份，删除该年的子进度文件后重新执行该年度检索导出
- 对只有 `YYYY-no-results.txt` 的年份直接跳过，不参与年度 Excel 校验与最终总合并
- 对缺少 `merged.xlsx`、缺少年度汇总报告或 `merged.xlsx` 无法读取的年份，记录到最终总报告并跳过，不再触发整轮自动续跑
- 只有全部年份校验一致，才继续最终总合并
- 为避免死循环，年度重跑最多执行 `2` 轮，超过上限则直接失败并阻止总合并
- 新增公共校验工具，统一复用年度报告定位、合并文件定位、报告总数解析与 Excel 行数统计能力
