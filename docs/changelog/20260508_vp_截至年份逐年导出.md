## 1. 变更摘要
为 `vp-search advanced-search` 同步接入“截至年份全量导出逐年编排”能力：

- 当存在 `--date-to` 且没有 `-n/--max-download` 时，切换到逐年模式，不再直接做单次全量分批导出。
- 逐年模式按维普结束年下拉中的真实可选年份生成年度任务。
- 无论是否传入 `--date-from`，每轮都显式把 `date_from/date_to` 写成同一年，按严格单年窗口执行。
- 传入 `--date-from` 时，它只决定从哪一年开始遍历，不再参与累计区间构造。
- 年度子任务继续复用现有批量导出主流程、子进度文件、断点续跑与结果合并能力。
- 复用同一高级检索页时，起始年和结束年都会被显式覆盖；空值会主动恢复到下拉默认项，避免沿用上一轮残留年份。
- 打开维普高级检索页失败时，异常类型从参数校验类调整为导航状态类，使 CLI 自动续跑可以接管这类瞬时页面异常。

## 4. 后续优化 (2026-05-09)

### 4.1 VP 逐年年份边界优化

**问题**：候选年份只从 1989 开始，但代码硬编码 1949 作为最小年份，导致无效任务。

**变更**：
- 新增常量 `YEARLY_VP_MIN_YEAR = 1989`
- `date_to < 1989` 时输出日志并直接退出程序
- `date_from < 1989` 时自动钳制到 1989（视为未传入）
- 第一年任务特殊处理：起始年钳制到 1989，终止年按当年可用年份
- `_run_single_yearly_export` 中 `reuse_current_search_page` 改为 `True`

**文件变更**：
- `vp-search/scripts/vp_yearly_export_ops.py`:
  - 第32行新增 `YEARLY_VP_MIN_YEAR = 1989`
  - `_build_yearly_export_tasks` 方法增加 VP 年份边界校验与首年特殊处理
  - `_run_single_yearly_export` 中 `reuse_current_search_page=False` → `True`
