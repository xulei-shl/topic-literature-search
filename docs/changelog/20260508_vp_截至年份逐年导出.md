## 1. 变更摘要
为 `vp-search advanced-search` 同步接入“截至年份全量导出逐年编排”能力：

- 当存在 `--date-to` 且没有 `-n/--max-download` 时，切换到逐年模式，不再直接做单次全量分批导出。
- 逐年模式按维普结束年下拉中的真实可选年份生成年度任务。
- 无论是否传入 `--date-from`，每轮都显式把 `date_from/date_to` 写成同一年，按严格单年窗口执行。
- 传入 `--date-from` 时，它只决定从哪一年开始遍历，不再参与累计区间构造。
- 年度子任务继续复用现有批量导出主流程、子进度文件、断点续跑与结果合并能力。
- 复用同一高级检索页时，起始年和结束年都会被显式覆盖；空值会主动恢复到下拉默认项，避免沿用上一轮残留年份。
- 打开维普高级检索页失败时，异常类型从参数校验类调整为导航状态类，使 CLI 自动续跑可以接管这类瞬时页面异常。

## 2. 文件清单
- `vp-search/scripts/vp_search_interactor.py`: 修改，接入逐年模式入口分流
- `vp-search/scripts/vp_yearly_export_ops.py`: 新增，负责逐年编排、总进度与总报告
- `vp-search/scripts/vp_yearly_progress_store.py`: 新增，负责逐年外层进度文件
- `vp-search/scripts/vp_form_ops.py`: 修改，复用页面时显式覆盖起始/结束年份，并修正高级检索页异常分类
- `vp-search/scripts/utils.py`: 修改，补充逐年模式可读输出
- `tests/test_vp_interactor.py`: 修改，新增逐年模式、单年窗口任务、续跑、年份清空与异常分类测试
- `tests/test_vp_progress_store.py`: 修改，新增逐年外层进度文件测试
