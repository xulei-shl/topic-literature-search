# 开发变更记录
- **日期**: 2026-05-06
- **对应设计文档**: 无，本次为已批准微小行为调整

## 1. 变更摘要
- 统一 `cnki-search` 与 `vp-search` 的默认结果输出目录。
- 默认产物现在统一写入仓库根目录下的 `outputs/<命名空间>/<检索词>`。
- 将两处重复的根目录解析与输出目录创建逻辑收敛到 `src/utils/output_path.py`。
- 继续收敛两套检索脚本的公共逻辑，新增共享的日期解析、结果保存、进度文件基类与配置基类。
- `cnki-search` 与 `vp-search` 现只保留站点差异化实现，公共规则统一由 `src/utils` 维护。

## 2. 文件清单

- `src/utils/progress_store.py`: 新增，统一进度文件读写、参数恢复与指纹计算逻辑。

- `cnki-search/scripts/cli.py`: 修改，改为复用公共日期解析逻辑。
- `cnki-search/scripts/config.py`: 修改，改为复用公共配置基类。
- `cnki-search/scripts/progress_store.py`: 修改，改为复用公共进度文件基类。
