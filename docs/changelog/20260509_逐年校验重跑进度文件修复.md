# 开发变更记录
- **日期**: 2026-05-09
- **对应设计文档**: (基于 [CNKI 截至年份逐年导出设计文档](/F:/Github/topic-literature-search/docs/design/cnki_date_to_yearly_export_20260508.md) 的进度文件修复)

## 1. 变更摘要
修复年度校验重跑时进度文件写入逻辑的潜在问题：

- 重跑循环中 snapshot 的 `next_year_index` 不再设置为已完成的年份索引，避免恢复时重跑已完成年份
- 新增 `runtime.validation_rerun_in_progress` 字段标记重跑状态，防止中断后误进入外层年度循环
- `_run_yearly_advanced_export` 入口读取该标记，如检测到重跑中断则跳过外层年度循环，直接进入校验重跑阶段
- `cleanup_year_output_dir` 增强：处理子目录时使用 `shutil.rmtree()`，避免 Windows 环境下 `OSError` 静默跳过导致旧文件残留

## 2. 文件清单
- `src/utils/yearly_export_validation.py`: 修改，`cleanup_year_output_dir` 增加子目录递归删除
- `wanfang-search/scripts/wanfang_yearly_export_ops.py`: 修改，`_save_yearly_progress_snapshot` 增加 `validation_rerun_in_progress` 参数及标记写入/读取逻辑
- `vp-search/scripts/vp_yearly_export_ops.py`: 同上
- `cnki-search/scripts/cnki_yearly_export_ops.py`: 同上

## 3. 进度文件兼容性
- 新增 `validation_rerun_in_progress` 字段，使用 `.get()` 安全读取，兼容旧进度文件
- 重跑完成后正常状态写入不含该字段（或值为 false），不影响后续正常续跑

## 4. 验证
- `python -m py_compile` 语法检查通过

## 5. 增量变更 (2026-05-11)
调整逐年校验的容差阈值，允许实际数量与报告数量相差 ≤10 条时视为成功，避免不必要重试：

- `_collect_yearly_validation_outcomes` 中 `if reported_total != actual_rows` 改为 `if abs(reported_total - actual_rows) > 10`
- 错误消息格式从 `{year}({reported}!={actual})` 改为 `{year}({diff}相差)`，更直观显示差值
- 涉及文件：`cnki-search/scripts/cnki_yearly_export_ops.py`、`vp-search/scripts/vp_yearly_export_ops.py`、`wanfang-search/scripts/wanfang_yearly_export_ops.py`
