# 开发变更记录
- **日期**: 2026-05-09
- **对应设计文档**: 无（调试修复）

## 1. 变更摘要
- 调整万方高级检索批次勾选流程，禁止页面勾选存在缺口时继续推进批次游标。
- 为万方批次勾选新增“重建检索结果上下文 -> 关闭浏览器重试当前批次”的逐级恢复链路。
- 调整整页全选策略：整页全选一旦无法确认成功，立即终止当前页并重建检索，不再回退逐条勾选。
- 增加整页全选元素的多条件就绪判断与多节点点击链路，优先减少首次全选失败。
- 保持共享导出骨架默认行为不变，仅增加站点可覆写的批次勾选钩子。

## 2. 文件清单
- `src/core/advanced_export_flow.py`: 修改，新增批次勾选钩子，允许站点层接管批次重试
- `wanfang-search/scripts/wanfang_selection_ops.py`: 修改，页面勾选未达标时直接失败，阻止缺口继续推进
- `wanfang-search/scripts/wanfang_selection_ops.py`: 修改，整页全选失败时直接交给上层重试，不再逐条补勾
- `wanfang-search/scripts/wanfang_selection_ops.py`: 修改，整页全选前先校验 DOM 结构就绪，并按 `label -> span -> inner -> input` 尝试勾选
- `wanfang-search/scripts/wanfang_search_interactor.py`: 修改，新增当前批次检索重建与浏览器级重试逻辑
- `tests/test_wanfang_interactor.py`: 修改，更新页面缺口预期并新增批次重试测试
