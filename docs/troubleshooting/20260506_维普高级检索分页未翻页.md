# 故障复盘报告
- **日期**: 2026-05-06
- **严重级别**: Medium

## 1. 问题现象
执行 `python vp-search/scripts/cli.py advanced-search --query "新青年" --date-to 2025 -n 100` 时，第一页 50 条文献勾选完成后没有继续翻到下一页，任务停留在第一页。

## 2. 根本原因 (Root Cause)
分页解析与翻页点击逻辑只依赖 `#headerpager`。当页面实际可用的“下一页”按钮出现在 `#footerpager`，或者顶部分页按钮处于禁用状态时，代码仍会优先拿到顶部定位器并提前返回，所以翻页流程被误判为“没有下一页可点”。

## 3. 解决方案
为维普结果页分页逻辑补齐 `#footerpager` 支持：

- 结果页解析同时识别 `#headerpager` 与 `#footerpager`
- 页码与总页数从两个分页容器中取最大有效值
- 查找“下一页”按钮时跳过禁用按钮，优先返回真正可点击的分页入口

## 4. 预防措施
- 新增回归测试，覆盖“顶部分页禁用、底部分页可用”场景
- 新增结果页解析测试，覆盖仅存在 `footerpager` 时的页码与总页数提取逻辑
