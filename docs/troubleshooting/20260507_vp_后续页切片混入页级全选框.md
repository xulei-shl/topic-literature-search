# 故障复盘报告
- **日期**: 2026-05-07
- **严重级别**: Medium

## 1. 问题现象
`vp-search` 在高级检索限定导出数量时，目标数量落在后续页的部分页场景中，可能出现最终选中数量大于计划导出数量的问题。

典型现象：
- 计划导出 `155`
- 实际选中/导出 `200`

## 2. 根本原因 (Root Cause)
`vp-search/scripts/vp_checkbox_list_ops.py` 为了提升后续页勾选性能，对“按页码切出且长度正好等于每页条数”的候选集合直接复用，没有先过滤掉页级全选复选框。

当维普结果页的复选框列表中混入 `selectArticleAll` 时：
- 当前页候选集合首项可能不是结果行，而是页级全选框
- 后续的逐条勾选逻辑会错误点击页级全选
- 最终导致本应只补勾少量记录，却整页额外选中了 50 条

## 3. 解决方案
在 `vp-search/scripts/vp_checkbox_list_ops.py` 中调整当前页复选框收集逻辑：
- 后续页命中整页切片时，先对当前切片做轻量过滤，剔除页级全选框
- 如果过滤后条数不足一页，优先顺延补齐后续结果行
- 仅在仍无法形成可信切片时，才回退到可见性扫描

这样既保留了局部切片的性能优势，又避免了页级全选框污染结果行集合。

## 4. 预防措施
- 新增测试覆盖“后续页切片混入页级全选框”场景，确保不会再误触发整页全选
- 保留“中间页切片不完整时回退可见性扫描”的测试，确保异常 DOM 结构下仍以准确性优先
