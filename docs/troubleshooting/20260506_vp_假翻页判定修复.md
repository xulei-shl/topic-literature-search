# 故障复盘报告
- **日期**: 2026-05-06
- **严重级别**: Medium

## 1. 问题现象
维普结果页在第一页选满后会进入翻页流程，但程序把“标题变化”误判为“翻页成功”，随后继续按第一页处理第二轮勾选，导致重复勾选或死循环。

## 2. 根本原因 (Root Cause)
因为旧逻辑只要检测到 URL、页码文本或首条标题任一变化，就会认为结果页已经发生切换。实际运行中，点击“下一页”后页面可能先出现局部刷新或中间态，标题变化了，但页码仍然停留在 `1/100`。

## 3. 解决方案
- 翻页时读取 `data-page` 作为目标页码
- 等待逻辑改为“当前页码必须推进到目标页”
- 即使检测到局部状态变化，只要页码未推进，也继续等待或重试

## 4. 预防措施
- 新增回归测试，覆盖“标题变化但页码未变”时不得判定为翻页成功的场景
- 后续涉及分页切换完成判定时，优先以页码推进为准，避免仅依赖标题或局部 DOM 变化
