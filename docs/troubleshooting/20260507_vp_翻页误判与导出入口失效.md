# 故障复盘报告
- **日期**: 2026-05-07
- **严重级别**: Medium

## 1. 问题现象
执行维普高级检索导出时，日志显示结果页已完成跨页勾选，但在进入导出阶段后报错 `未找到"导出题录"入口`。同一轮日志中还出现了翻页后页码与勾选上下文不一致的现象，例如翻页完成日志指向目标页，而后续勾选循环中的 `current_page` 又回退为旧页码。

## 2. 根本原因 (Root Cause)
因为结果页翻页逻辑将“首条结果标题变化”误判为“页码已经推进”，所以在分页栏尚未刷新完成时，内部状态提前把当前页标记为目标页，造成真实 DOM 页码与内部页码状态脱节。随后勾选链路一部分逻辑读取解析页码，另一部分逻辑读取最近已知页码，进一步放大了跨页状态不一致的问题。

另一个独立问题是导出入口定位过于依赖固定选择器。因为“导出题录”入口可能出现在“批量处理”展开后的弹层或下拉菜单容器中，而原逻辑只在主页面结构下查找固定节点，所以菜单明明已经展开，仍然会错误报出入口不存在。

## 3. 解决方案
收紧翻页成功判定，只在解析到结果页页码真实推进后才认定翻页完成，不再把标题变化作为成功条件。同时统一勾选链路中的当前页解析，优先复用已确认的页码解析逻辑，避免同一轮流程中出现页码分裂。

导出阶段为“批量处理”展开后的菜单增加了额外的入口选择器，并在菜单展开后基于合并后的选择器集合继续等待和点击“导出题录”入口，提升弹层场景下的兼容性。

## 4. 预防措施
新增回归测试，覆盖“标题变化但页码未推进时不得认定翻页成功”以及“批量处理菜单展开后通过菜单选择器继续点击导出入口”两个场景。后续若继续调整维普翻页或导出逻辑，必须先保留这两类测试，再进行重构。
