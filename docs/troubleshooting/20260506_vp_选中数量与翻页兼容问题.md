# 故障复盘报告
- **日期**: 2026-05-06
- **严重级别**: Medium

## 1. 问题现象
执行维普高级检索批量导出时，第一页勾选后没有继续翻页，且页面显示的“已选择数量”与程序内部判断不一致。

## 2. 根本原因 (Root Cause)
因为结果页前端 DOM 已发生变化，代码仍在使用旧版选择器读取已选数量与分页控件：

- 已选数量实际显示在 `.checked-tip .mark-number`，但代码仍读取旧节点
- “下一页”控件不再稳定暴露为 `.layui-laypage-next`，仅依赖旧选择器会漏掉真实可点击按钮
- “每页 50 条”入口存在容器实现差异，旧代码只匹配固定路径

因为这些旧选择器未命中，所以程序虽然完成了勾选动作，但后续的选中数量判断、分页判断和分页尺寸切换都可能失效，最终表现为没有继续翻页。

## 3. 解决方案
- 将已选数量解析扩展为优先读取 `.checked-tip .mark-number`
- 为“下一页”按钮增加基于文本的宽松匹配逻辑，兼容 `a`、`button`、`span` 容器
- 为“每页 50 条”入口增加 `data-count='50'` 的通用兜底选择器
- 保持非整页目标逐条勾选，避免例如 `-n 40` 时误触发整页全选

## 4. 预防措施
- 新增回归测试，覆盖新版已选数量节点
- 新增回归测试，覆盖通用 `data-count='50'` 的分页尺寸入口
- 新增回归测试，覆盖仅靠“下一页”文本即可识别翻页按钮的场景
