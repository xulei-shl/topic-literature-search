# 故障复盘报告
- **日期**: 2026-05-07
- **严重级别**: Medium

## 1. 问题现象

执行 `vp-search` 高级检索批量导出时，翻到第 7 页后出现批量勾选异常：

- 日志显示当前页复选框被错误识别为 `3` 个，随后误触发整页全选。
- 页级全选后日志出现 `expected_increase=3, actual_increase=50`。
- 第 8 页开始按页码切片失效，回退到全量可见性扫描，单页等待耗时接近两分钟。

## 2. 根本原因 (Root Cause)

- `vp-search/scripts/vp_checkbox_list_ops.py` 使用“全局 checkbox 列表按 `current_page * page_size` 切片”的假设来识别当前页结果。
- 维普结果页的 checkbox 节点总数并不与翻页后的结果行严格线性对应，因此第 7 页切片后只落到尾部 `4` 个节点，过滤后只剩 `3` 个结果行。
- `vp-search/scripts/vp_selection_ops.py` 仅依据 `page_target_count == row_count` 判断是否允许整页全选，没有识别出这是一个不完整样本，于是触发了真正的整页 50 条全选。
- 第 8 页时切片起点越界，逻辑退化为对 300+ 个节点逐个做可见性探测，导致明显卡顿。

## 3. 解决方案

- 调整 `vp-search/scripts/vp_checkbox_list_ops.py`：
  - 中间页切片结果若不足一整页，判定为不可信，改走可见结果回退路径。
  - 回退路径改为边扫描边收集，找到当前页 `page_size` 个可见结果后立即停止，避免扫完整个全局节点集。
- 调整 `vp-search/scripts/vp_selection_ops.py`：
  - 只有当当前页候选数量达到当前分页大小时，才允许触发整页全选。
  - 当页面对象不完整或读取分页大小失败时，回退到默认分页大小保护。

## 4. 预防措施

- 新增 `tests/test_vp_interactor.py` 用例，覆盖：
  - 中间页切片不完整时必须回退到可见结果采集。
  - 候选数小于分页大小时禁止触发整页全选。
- 保留共享骨架与维普站点定向回归：
  - `pytest tests/test_vp_interactor.py -q`
  - `pytest tests/test_advanced_export_flow.py tests/test_vp_interactor.py -q`
