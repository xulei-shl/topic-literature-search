# 故障复盘报告
- **日期**: 2026-05-04
- **严重级别**: Medium

## 1. 问题现象
清理后的批次 Excel 里仍然能看到重复表头行，且表格列名显示为 `0, 1, 2 ...`，随后在参考格式回填时继续出现行数不一致报错。

## 2. 根本原因 (Root Cause)
原始导出表格在读取后，并没有把 `SrcDatabase-...`、`Title-...`、`Author-...` 这一行识别为真正表头，而是把它当作普通数据行保留，导致 `DataFrame` 列名仍是数字序号。此前“按列表头删除重复表头”的逻辑因此失效，重复表头无法被正确清理。

## 3. 解决方案
在 [cnki-search/scripts/export_processor.py](/E:/Desk/xinqingnian/cnki-search/scripts/export_processor.py:18) 增加表头规范化步骤：先扫描真实表头行，将其提升为 `DataFrame` 列名，再删除该行以及后续任意位置再次出现的重复表头行。表头识别改为基于稳定前缀 `SrcDatabase-`、`Title-`、`Author-`，避免依赖整行完全相等。

## 4. 预防措施
后续处理第三方导出表格时，先判断“表头是否已经进入列名”，再决定删除策略。对于列名仍为数字序号的表格，必须先执行表头提升，不能直接按现有列名做重复表头过滤。
