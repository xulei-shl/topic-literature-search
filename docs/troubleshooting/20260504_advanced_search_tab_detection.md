# 故障复盘报告
- **日期**: 2026-05-04
- **严重级别**: Medium

## 1. 问题现象
执行 `python cnki-search/scripts/cli.py advanced-search --query "大模型 知识服务" -n 100` 时，浏览器实际已打开高级检索页面，但脚本报错“点击首页‘高级检索’后未进入高级检索页面”。

## 2. 根本原因 (Root Cause)
高级检索入口逻辑使用 URL 是否包含 `advanced` 或 `advsearch` 作为成功判定条件。由于知网页面可能在新标签页中打开，且实际 URL 不一定满足该字符串匹配规则，所以页面虽然已经正常打开，脚本仍会误判为失败。

## 3. 解决方案
将高级检索页面判定逻辑改为检测目标页面元素是否出现，不再依赖 URL 字符串。点击首页“高级检索”后，轮询当前页和新打开的标签页；一旦在某个页面中检测到 `#gradetxt`、`#datebox1`、`input.btn-search` 或 `#txt_1_value1` 等高级检索特征元素，即切换当前活动页并继续执行后续流程。

## 4. 预防措施
后续处理多标签页场景时，优先使用稳定的页面业务元素识别目标页，避免将 URL 字符串匹配作为唯一判定条件。当前已补充最小语法校验，后续建议增加一次真实页面联通验证。
