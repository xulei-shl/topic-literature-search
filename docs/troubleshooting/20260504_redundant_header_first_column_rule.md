# 故障复盘报告
- **日期**: 2026-05-04
- **严重级别**: Medium

## 1. 问题现象
批次元数据文件经过清理后，仍然存在少量重复表头行，导致参考格式 TXT 条数与清理后 Excel 数据行数仍然相差 1 行。

## 2. 根本原因 (Root Cause)
此前的重复表头识别逻辑仍然带有结构推断，依赖表头提升和多列特征匹配。在部分导出文件里，重复表头虽然本质明确，但结构并不完全稳定，导致个别重复表头未被识别和删除。

## 3. 解决方案
将重复表头删除规则收敛为更直接的业务规则：只要某一行第一列的值以 `SrcDatabase` 开头，就将该行视为重复表头并删除。该规则已实现在 [cnki-search/scripts/export_processor.py](/E:/Desk/xinqingnian/cnki-search/scripts/export_processor.py:160)。

## 4. 预防措施
对于第三方导出文件的清理逻辑，优先采用稳定、可观察、可解释的业务规则，避免过度依赖结构推断。若后续发现新的噪声行特征，再按同样方式补充明确规则。
