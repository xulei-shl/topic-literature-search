# 故障复盘报告
- **日期**: 2026-05-06
- **严重级别**: Medium

## 1. 问题现象
维普结果页第一页实际只展示 50 条，但程序在批量勾选日志中将当前页记录数识别为 304，导致选满第一页后仍在当前页继续尝试勾选更高下标，而不是翻页。

## 2. 根本原因 (Root Cause)
因为维普页面中的原始 `input[type='checkbox']` 本身常处于隐藏状态，真正可操作的是旁边的 Layui 包装层。仅依赖 `input:visible` 或全量 `count()` 都无法可靠区分“当前页可操作项”和“隐藏的跨页节点”。

## 3. 解决方案
- 基于全量复选框集合逐个判断可见性
- 若原始 input 不可见，则继续检查其对应的 Layui 包装层是否可见
- 仅将可见项纳入当前页记录数、逐条勾选与翻页判断

## 4. 预防措施
- 新增回归测试，覆盖“全量 304 个复选框、当前页仅 50 个可见”的场景
- 后续涉及维普结果列表的可操作项判断时，优先以真实可点击节点可见性为准，而不是仅依赖原始 input 的显示状态
