# 故障复盘报告
- **日期**: 2026-05-04
- **严重级别**: Medium

## 1. 问题现象
高级检索导出流程虽然能在内存中跳过重复表头行完成后续处理，但输出目录中的元数据 Excel 文件本身仍然保留了这些重复表头，用户打开文件后看到脏数据未被删除。

## 2. 根本原因 (Root Cause)
之前的修复只在回填“参考格式”列前，对内存中的 `DataFrame` 做了重复表头过滤，并没有把过滤结果重新保存到导出的 Excel 文件中。因此业务校验可能通过，但最终落盘文件仍然是原始下载的脏 `xls`。

## 3. 解决方案
在 [cnki-search/scripts/export_processor.py](/E:/Desk/xinqingnian/cnki-search/scripts/export_processor.py:18) 新增导出表格清理并落盘的方法。高级检索批次流程在下载原始元数据 `xls` 后，立即生成清理后的 `metadata-cleaned.xlsx`，删除原始脏文件，并让后续参考格式回填与最终合并都基于该清理后的文件执行。

## 4. 预防措施
后续处理第三方下载文件时，区分“内存中处理成功”和“最终输出文件已修正”两个层面。对于用户会直接打开查看的中间产物，必须确保清理结果已真实写回磁盘文件。
