# 故障复盘报告
- **日期**: 2026-05-06
- **严重级别**: Medium

## 1. 问题现象
执行 `python vp-search/scripts/cli.py advanced-search --query "新青年" --date-to 2025 -n 50` 时，高级检索在导出阶段失败，报错为“不支持的表格格式”。失败现场生成的 `metadata.txt` 与 `reference.txt` 内容一致，均为参考文献文本，而非 Excel 文件。

## 2. 根本原因 (Root Cause)
导出逻辑在点击 `Excel导出` 选项后，没有校验页面是否真正切换到 `excel` 选中态，也没有捕获“点击导出类型即自动下载”的页面行为。结果代码继续点击确认按钮时，页面仍可能停留在 `参考文献` 导出态，所以 `metadata` 分支下载到的是 `.txt` 文本文件，后续表格解析阶段因此失败。

## 3. 解决方案
在 `vp-search/scripts/interactor.py` 中调整导出流程：

- 优先捕获点击导出类型时触发的自动下载。
- 若点击导出类型后未直接下载，则等待对应类型进入 `layui-this` 选中态，再回退到确认按钮下载。
- 对 `metadata` 下载结果增加后缀校验，若不是 Excel 文件则直接报出“Excel 导出未生效”。

同时在 `tests/test_vp_interactor.py` 中补充回归测试，覆盖“自动下载”“回退确认按钮”“误下为 txt 时拦截失败”三条路径。

## 4. 预防措施
- 保留新增单元测试，防止后续重构再次忽略页面的自动下载行为。
- 遇到多格式导出页面时，统一先验证选中态或下载结果，再进入后续文件解析逻辑。
