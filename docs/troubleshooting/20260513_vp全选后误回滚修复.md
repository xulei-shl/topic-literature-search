# 故障复盘报告
- **日期**: 2026-05-13
- **严重级别**: High

## 1. 问题现象
维普结果页在执行“全选 -> 翻页 -> 全选”循环时，第一页明明已经勾选成功，但程序没有进入翻页，而是立即开始逐条取消勾选，并在日志中持续出现 `Locator.uncheck: Element is not visible`。

## 2. 根本原因 (Root Cause)
全选后的验收逻辑过于激进。因为页面使用 Layui + AJAX，`已选数量` 与单条复选框状态不会在点击全选后瞬时全部稳定，但代码在点击全选后立刻读取一次数量；一旦短暂读到的增量小于预期，就直接把当前页判定为“全选失败”，并调用整页回滚逻辑。由于维普原生 `input` 通常不可见，回滚阶段对隐藏 input 执行 `uncheck()`，于是产生了大量“元素不可见”的误报。

## 3. 解决方案
- 为全选验收增加短轮询等待，先等待页内勾选状态和已选计数稳定，再判断是否达标。
- 当全选后的数量暂未达标时，不再整页清空，而是回退到逐条补勾缺口。
- 取消勾选优先点击可见的 Layui 包装层，再回退到原生 `input` 与 JS 兜底，避免直接对隐藏 input 执行 `uncheck()`。

## 4. 预防措施
- 新增测试覆盖“全选短暂不足时不整页回滚”和“取消勾选优先使用可见包装层”两个场景。
- 之后所有基于 AJAX 的批量勾选页面，都应遵循“等待稳定后验收、失败先补缺口、不做整页回滚”的策略。
