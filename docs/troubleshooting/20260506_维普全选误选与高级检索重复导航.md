# 故障复盘报告
- **日期**: 2026-05-06
- **严重级别**: Medium

## 1. 问题现象
执行 `python vp-search/scripts/cli.py advanced-search --query "新青年" --date-to 2025 -n 100` 时，出现两类异常：

- 第一页 50 条勾选后未继续翻页
- 某些运行会在进入高级检索前直接报错：`Navigation to "https://qikan.cqvip.com/" is interrupted by another navigation to "https://qikan.cqvip.com/"`

## 2. 根本原因 (Root Cause)
因为维普结果页的 `selectArticleAll` 在部分场景下并不严格等同于“仅全选当前页”，所以当代码按“当前页 50 条”假设使用它时，已选数量可能一次性扩大到跨页范围。这样一来，批量勾选循环会误判“本批次目标数量已经满足”，所以不会继续执行翻页。

因为高级检索入口点击后本身就可能触发页面跳转，所以在 `restore_session()` 返回首页后再点击“高级检索”，紧接着立即执行一次固定 `goto(advanced_search_url)`，会形成重叠导航，最终触发 Playwright 的导航中断异常。

## 3. 解决方案
- 仅在批次第一页尝试页级全选
- 页级全选后立即校验“已选数量增量”是否等于当前页目标数量
- 如果增量异常，则清空本次异常选择并回退到逐条勾选
- 点击高级检索入口后，先等待页面稳定并检查是否已经进入目标页；只有仍未进入时才执行兜底 `goto`
- 调整高级检索入口流程，避免向 `restore_session()` 传入同一首页地址造成重复首页导航

## 4. 预防措施
- 新增回归测试，覆盖“页级全选数量异常时回退逐条勾选”场景
- 新增回归测试，覆盖“点击高级检索入口后已进入目标页，不再二次 goto”场景
- 后续若继续增强批量导出，优先保留“选择结果增量校验”这类防误选保护，避免仅依赖前端控件名称判断语义
