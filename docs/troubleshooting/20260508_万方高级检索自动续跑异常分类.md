# 故障复盘报告
- **日期**: 2026-05-08
- **严重级别**: Medium

## 1. 问题现象
- 对照 `cnki` 检查 `wanfang` 高级检索逐年导出链路时，需要确认两个问题是否同样存在：
  - 逐年执行后高级检索页签是否会持续堆积
  - 打开高级检索页失败时，是否会因为异常分类错误而无法自动续跑

## 2. 根本原因 (Root Cause)
- 页签堆积问题在 `wanfang` 中未复现，因为 `wanfang-search/scripts/wanfang_form_ops.py` 的 `_open_advanced_search_page()` 直接调用 `restore_session(self.config.advanced_search_url)` 在当前页打开高级检索 URL，并没有像 `cnki` 那样通过点击首页入口触发潜在的新页签。
- 自动续跑问题在 `wanfang` 中存在，因为 `_open_advanced_search_page()` 在页面未就绪时抛出的是 `ValidationError`，所以 `wanfang-search/scripts/cli.py` 里的 `run_with_auto_resume(...)` 会把它当成不可重试异常直接中断，而不是进入自动续跑。

## 3. 解决方案
- 保持页签打开路径不变，不对 `wanfang` 增加多余的页签切换/清理逻辑。
- 将 `wanfang-search/scripts/wanfang_form_ops.py` 中“打开万方高级检索页面失败”的异常类型从 `ValidationError` 调整为 `NavigationStateError`，使其归类为可重试的瞬时页面状态异常。

## 4. 预防措施
- 新增测试覆盖：
  - `tests/test_wanfang_interactor.py`：验证万方高级检索通过 `advanced_search_url` 在当前页打开，不存在 `cnki` 式新开 tab 链路。
  - `tests/test_wanfang_interactor.py`：验证高级检索页未就绪时抛出 `NavigationStateError`，确保 CLI 自动续跑可接管。
- 验证命令：
  - `python -m pytest tests/test_wanfang_interactor.py tests/test_advanced_search_retry.py tests/test_site_cli_auto_resume.py -q`
  - `python -m compileall wanfang-search/scripts`
