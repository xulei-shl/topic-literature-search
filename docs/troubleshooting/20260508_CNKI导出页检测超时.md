# 故障复盘报告
- **日期**: 2026-05-08
- **严重级别**: Medium

## 1. 问题现象
CNKI 高级检索批量导出执行到中途时，在打开“自定义”导出页后报错：

`Page.reload: Timeout 80000ms exceeded`

异常发生在 `cnki-search/scripts/cnki_export_ops.py` 的导出页就绪检测逻辑，任务因此中断，无法继续导出后续批次。

## 2. 根本原因 (Root Cause)
因为导出逻辑在新页面打开后会先执行一次 `export_page.reload(wait_until="commit")`，所以代码把“页面是否可用”的判断绑定到了新的导航提交事件上。

但 CNKI 实际页面在很多情况下已经渲染出左侧“文献导出格式”栏和“自定义”标签，不再需要额外刷新，也未必会再触发新的导航提交。此时页面本身可见、可操作，但 `reload` 仍会继续等待导航，最终触发超时。

另外，原有就绪检测只盯着 `.check-labels`，对当前导出页结构覆盖不足，导致页面已经进入导出态时仍可能被误判为“未就绪”。

## 3. 解决方案
本次修复对 CNKI 导出页检测做了最小化调整：

1. 移除进入导出页后的前置强制 `reload`。
2. 将导出页就绪判定改为轮询多个稳定页面标记，包括：
   - `.export-sidebar-a`
   - `li.current a[displaymode='selfDefine']`
   - `#litoexcel`
   - `#litotxt`
   - `.check-labels`
3. 先等待 `domcontentloaded`，再执行元素级等待，避免把元素可见问题误判成导航超时。

## 4. 预防措施
已新增 CNKI 单元测试，覆盖以下场景：

- 导出页出现左侧格式栏时应直接判定为就绪。
- 导出页已打开时不应再默认触发 `reload()`。

后续若 CNKI 再次调整导出页 DOM，应优先补充“真实可见标记”选择器，而不是恢复强制刷新策略。
