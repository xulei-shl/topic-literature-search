# 故障复盘报告
- **日期**: 2026-05-06
- **严重级别**: Medium

## 1. 问题现象
执行维普高级检索批量导出时，第一页勾选完成后没有翻到下一页，而是在当前页继续尝试勾选更大的下标，最终卡在不可见复选框上。

## 2. 根本原因 (Root Cause)
因为结果页 DOM 中同时存在大量隐藏的跨页复选框节点，代码在批量勾选时直接使用全量复选框集合计数，所以把 `304` 个隐藏/跨页节点误判成“当前页共有 304 条记录”。

因为当前页真实可见记录只有 `50` 条，代码在第一页选完 50 条后仍然认为当前页还有剩余记录，因此继续尝试勾选 `nth(100+)` 这类不可见节点，而没有进入翻页分支。

## 3. 解决方案
- 新增“当前页可见复选框”定位逻辑，优先使用 `:visible` 过滤结果复选框
- 批量勾选与当前页逐条勾选统一改为基于可见复选框集合工作
- 保留原有全量复选框定位逻辑，避免影响其他非分页场景

## 4. 预防措施
- 新增回归测试，覆盖“全量 DOM 复选框远大于当前页可见复选框”时仍应正确翻页的场景
- 后续涉及分页列表选择时，优先区分“当前页可见节点”与“全量匹配节点”，避免再次把缓存/隐藏节点计入业务数量
