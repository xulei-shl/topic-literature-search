# 故障复盘报告
- **日期**: 2026-05-04
- **严重级别**: Medium

## 1. 问题现象
执行高级检索时，浏览器已经打开知网页面，但脚本在配置检索条件阶段报错“高级检索下拉项不存在”。

## 2. 根本原因 (Root Cause)
高级检索条件的字段下拉框使用了瞬时判空逻辑。因为知网页面在进入高级检索后可能继续刷新，或下拉项需要在点击触发器后异步渲染，所以脚本在下拉项尚未可见时就直接判定为不存在并抛错。

## 3. 解决方案
将高级检索下拉选择逻辑改为基于 `page_timeout` 的动态等待与重试：先等待下拉触发器可见并点击，再等待目标下拉项可见后执行点击；若在超时时间内始终未出现，再抛出校验异常。

## 4. 预防措施
为高级检索下拉选择补充回归测试，覆盖“首次不可见、稍后出现”的时序场景，以及“超时后抛错”的异常路径，避免后续改动重新引入瞬时判空问题。
