# 维普文献检索爬虫 - 模拟操作完整说明

## 一、模块概述

`vp-search` 是 `topic-literature-search` 项目中专门用于抓取**维普期刊数据库**（https://qikan.cqvip.com/）的自动化检索模块。

### 核心能力
- ✅ 高级检索（题名/关键词/摘要/年份/核心期刊）
- ✅ 批量导出元数据（Excel）与参考文献（TXT）
- ✅ 断点续传（意外中断可恢复）
- ✅ 逐年全量导出（时间跨度大可自动分段）
- ✅ 兜底重试机制（数据不一致时自动重跑）
- ✅ 会话持久化（登录状态保存复用）

---

## 二、技术架构摘要

### 核心技术
- **Playwright + Camoufox**：浏览器自动化，具备强反爬能力
- **Mixin 多继承**：功能模块解耦（表单、导航、导出、进度等）
- **模板方法模式**：`BaseAdvancedExportFlow` 定义骨架流程

## 三、逐页批量选择逻辑（循环操作）

这是爬虫的**核心循环**，用于逐页勾选结果并分批导出。

### 代码逻辑

```python
# 位于 BaseAdvancedExportFlow 和 VpSelectionMixin 中

page = 1
selected_total = 0
batch_index = 1

while page <= total_pages and selected_total < max_download:
    # Step 1: 确保每页 50 条（第一页时执行一次）
    if page == 1:
        set_page_size(50)  # 通过 #selectPageSize 下拉设置

    # Step 2: 等待结果页就绪
    wait_for_results_ready()

    # Step 3: 勾选当前页所有复选框
    checkboxes = page.locator("input[name='selectArticle']")
    current_page_count = checkboxes.count()

    for i in range(current_page_count):
        cb = checkboxes.nth(i)
        if not cb.is_checked():
            cb.check()
            selected_total += 1

    print(f"第 {page} 页：勾选 {current_page_count} 条，累计 {selected_total} 条")

    # Step 4: 判断是否达到批次上限（500 条）
    if selected_total >= EXPORT_BATCH_SIZE:
        print(f"✓ 批次 {batch_index} 已完成，启动导出...")
        export_selected_results(batch_index)
        batch_index += 1
        selected_total = 0  # 重置本批次计数器
        # 注意：页面上的复选框仍保持选中状态，用于累计

    # Step 5: 翻到下一页
    if has_next_page():
        click_next_page()
        page += 1
        wait_for_results_ready()
    else:
        break

# 循环结束后，导出最后一批（不足 500 条）
if selected_total > 0:
    export_selected_results(batch_index)
```

### 逐页翻页的三重降级策略

当“下一页”按钮不可用时，尝试以下方法：

| 优先级 | 方法 | 实现位置 |
|--------|------|---------|
| 1 | **页码输入框跳转**：输入目标页码 → 点击确定 | `_jump_to_results_page()` |
| 2 | **数字页码链接**：直接点击具体的页码数字 | `_goto_results_page_by_link()` |
| 3 | **点击下一页按钮**：最传统的翻页方式 | `_goto_next_results_page()` |

每页等待时间约 0.5 秒轮询检测状态变化。

## 四、分批导出机制

### 为什么需要分批？

**维普限制**：单次导出最多 500 条记录。

### 分批策略

```
总结果：1200 条

批次 1：第 1-10 页（每页 50 条 × 10 = 500 条） → 导出
批次 2：第 11-20 页（500 条） → 导出
批次 3：第 21-24 页（200 条） → 导出（最后一页）

导出文件：
  batch001/
    ├── metadata-batch001.xls
    └── reference-batch001.txt
  batch002/
    ├── metadata-batch002.xls
    └── reference-batch002.txt
  ...
  最终合并：merged.xlsx
```

### 导出流程（每批次）

```
Step 1: 点击"导出题录" →
   检测新页面打开（export_page）

Step 2: 切换为"表格"格式 (excel) →
   触发自动下载（Excel 文件）

Step 3: 切换为"参考文献"格式 (abstract) →
   触发自动下载（TXT 文件）

Step 4: 关闭导出页，返回结果页

Step 5: 保存批次元数据
```

## 五、逐年导出模式（Yearly Mode）

### 触发条件

```python
def _should_use_yearly_full_export(date_to, max_download):
    # 仅当指定了结束年份且未限制下载数量时触发
    return bool(date_to) and max_download is None
```

**典型场景**：
- 时间跨度 > 5 年
- 不限制 `-n` 参数
- 目标：获取某一主题在长时间范围内的**全部**文献

### 执行流程

```
用户请求：--query "人工智能" --date-from 2010 --date-to 2024

1. 收集可选年份
   → 访问高级检索页，读取结束年下拉框的所有年份

2. 生成任务列表
   → 2010, 2011, 2012, ..., 2024 共 15 个任务

3. 循环执行每年任务
   for year in [2010, 2011, ..., 2024]:
       搜索: 人工智能 + 年份 = year
       导出: 分批次下载 Excel + TXT
       保存: year-{year}/ 目录

4. 每 5 年重启浏览器
   → 避免反爬（ANTI_CRAWL_WAIT_INTERVAL = 5）
   → 随机等待 15-25 秒

5. 全部完成后，进入兜底校验阶段
```

## 六、兜底重试机制（重点）

### 目的
确保每个年份的导出结果**完整且一致**，防止因网络中断、导出失败导致数据缺失。

### 校验时机
**所有年份执行完毕后**，在最终合并 Excel 之前。

### 校验流程（三阶段）

#### 阶段 1：检查输出目录有效性

```python
if not _is_year_output_valid(year_output_dir):
    # 文件夹不存在或为空 → 加入 failures（重跑）
    failures.append({
        "year": "2020",
        "reason": "年份文件夹不存在" 或 "年份文件夹为空"
    })
    continue
```

#### 阶段 2：检查核心文件

```python
# 2.1 无结果标记（跳过校验，不重跑）
if find_yearly_no_results_note(year_output_dir):
    continue  # 该年确实无结果，正常

# 2.2 报告文件缺失 → 重跑
if report_path is None:
    failures.append({"reason": "缺少年度汇总报告文件"})
    continue

# 2.3 合并 Excel 缺失 → 重跑
if merged_excel_path is None:
    failures.append({"reason": "缺少年度合并 Excel 文件"})
    continue
```

#### 阶段 3：数据一致性校验

```python
try:
    reported_total = read_report_total(report_path)   # 从 JSON 读取报告的 total 字段
except Exception as exc:
    failures.append({"reason": f"年度汇总报告无法解析: {exc}"})
    continue

try:
    actual_rows = count_excel_rows(merged_excel_path)  # 读取 Excel 实际数据行数
except Exception as exc:
    failures.append({"reason": f"年度合并 Excel 无法读取: {exc}"})
    continue

# 对比：允许 ±10 行误差（可能因去重等处理）
if abs(reported_total - actual_rows) > 10:
    failures.append({
        "year": "2021",
        "reported_total": 500,
        "actual_rows": 450,
        # reason 自动取 item.get('reason', '数量不一致')
    })
```

### failures vs skipped_year_details

| 列表 | 含义 | 后续动作 | 何时加入 |
|------|------|----------|---------|
| `failures` | 校验失败，**需要重跑** | 清理目录 → 重新执行该年导出 | 文件缺失、数据不一致、解析异常 |
| `skipped_year_details` | 最终应该**跳过合并**的年份 | 不参与最终汇总 | 无结果年份、重试后仍失败 |

### 重试循环

```python
retry_round = 0
while True:
    failures, skipped_year_details = _collect_yearly_validation_outcomes()

    if not failures:
        # 所有年份都校验通过
        break

    retry_round += 1
    if retry_round > YEARLY_VALIDATION_MAX_RETRIES (2):
        # 超过最大重试次数 → 将 failures 转为 skipped
        raise ValidationError(f"逐年导出结果校验失败: {mismatch_summary}")

    logger.info(f"开始第 {retry_round} 轮重跑: {years}")

    for failure in failures:
        # 1. 清理该年目录
        cleanup_year_output_dir(output_dir, task)

        # 2. 重新执行单年导出（_run_single_yearly_export）
        rerun_result = ...

        # 3. 更新文件列表
        if rerun_final_file:
            yearly_result_files.append(rerun_final_file)

    # 继续下一轮校验...
```

### 重试限制

- **最大轮次**：`YEARLY_VALIDATION_MAX_RETRIES = 2`
- **含义**：每个失败年份最多重跑 2 次（原执行 + 2 次重试 = 共 3 次）
- **最终失败**：超过 2 次仍失败 → 抛 `ValidationError`，程序终止

## 七、具体操作步骤（按年份执行）

假设运行命令：
```bash
python vp-search/scripts/cli.py advanced-search \
    --query "人工智能" \
    --date-from 2010 \
    --date-to 2012
```

### 步骤 1：初始化
1. 创建输出目录 `outputs/人工智能/`
2. 创建年份子目录 `year-2010/`、`year-2011/`、`year-2012/`
3. 初始化外层进度文件 `outputs/人工智能/.vp-search/progress/人工智能.json`

### 步骤 2：执行 2010 年
```
a. 打开高级检索页
b. 填写：query="人工智能", date_from="2010", date_to="2010", core_only=False
c. 提交检索 → 等待结果
d. 逐页勾选（每页 50 条）
e. 每勾选满 500 条 → 导出一次（xls + txt）
f. 完成所有页 → 合并批次文件 → year-2010/2010-merged.xlsx
g. 写入 year-2010/2010-report.json
h. 记录到进度文件：executed_years = ["2010"]
```

### 步骤 3：执行 2011 年
```
复用当前高级检索页（reuse_current_search_page=True）
a. 刷新检索页（_reset_page_for_next_year）
b. 填写：date_from="2011", date_to="2011"
c. 重复批量选择、导出、合并
d. 生成 year-2011/2011-merged.xlsx
e. 记录：executed_years = ["2010", "2011"]
```

### 步骤 4：执行 2012 年
```
同理，生成 year-2012/2012-merged.xlsx
executed_years = ["2010", "2011", "2012"]
```

### 步骤 5：兜底校验（_rerun_years_until_export_consistent）
```
对每个年份 (2010, 2011, 2012):
    检查 year-2010/ 目录有效性
        ├─ 不存在 → failures (重跑)
        ├─ 为空   → failures (重跑)
        └─ 有效   → 继续

    检查 2010-no-results.txt
        ├─ 存在 → 该年无结果，跳过（不加入 failures）
        └─ 不存在 → 继续

    检查文件：
        ├─ report.json 缺失 → failures (重跑)
        ├─ merged.xlsx 缺失 → failures (重跑)
        └─ 都存在 → 继续

    解析数据：
        ├─ report.json 解析失败 → failures (重跑)
        ├─ merged.xlsx 读取失败 → failures (重跑)
        └─ 成功 → 对比

    对比：
        ├─ | reported_total - actual_rows | > 10
        │   → failures (数量不一致，重跑)
        └─ ≤ 10 → 校验通过

收集 failures = []
如果 failures 为空 → 进入最终合并
如果 failures 不为空 → 进入重试循环
```

### 步骤 6：重试循环（如有 failures）

假设 `year-2011` 的 `merged.xlsx` 只有 450 行，但 report.json 写的是 500 行（差 50 > 10）。

```
Round 1:
    failures = [{"year": "2011", "reason": "数量不一致(差50)"}]
    → 清理 year-2011/ 目录（删除所有旧文件）
    → 重新执行 _run_single_yearly_export("2011")
        → 再次检索 2011 年，重新导出
        → 生成新的 year-2011/ 文件
    → 再次校验 2011
    → 仍不一致？ → 继续 Round 2

Round 2:
    再次重跑 2011
    → 如果仍失败（超过 2 轮）→ 抛出异常，程序终止

如果所有 failures 都通过 → 跳出循环
```

### 步骤 7：最终合并

```python
# 仅合并校验通过的年份
yearly_result_files = [
    "year-2010/2010-merged.xlsx",   # 通过
    "year-2011/2011-merged.xlsx",   # 重试后通过
    "year-2012/2012-merged.xlsx",   # 通过
]

final_file = outputs/人工智能/20260511-210000-人工智能-merged.xlsx
merged = export_processor.merge_batch_excels(yearly_result_files, final_file)
```

### 步骤 8：写入总报告

```text

内容示例：
  检索词: 人工智能
  状态: success
  模式: yearly
  总数: 1500
  执行年份: 2010, 2011, 2012
  无结果年份: （空）
  跳过年份: （空）
  文件: 20260511-210000-人工智能-merged.xlsx
```

## 九、兜底重试的实际案例

### 场景 A：某年导出中断，文件缺失

**现象**：
```
year-2021/
  ├── batch001/              # 只导出了第一批
  │   ├── metadata-batch001.xls
  │   └── reference-batch001.txt
  └── 2021-report.json       # ❌ 不存在
```

**校验过程**：
1. `_is_year_output_valid` → ✅ 目录存在且有文件
2. 检查 `2021-no-results.txt` → ❌ 不存在（说明不是无结果）
3. 查找 `report.json` → ❌ 为 `None`
4. 判断结果：`failures.append({"reason": "缺少年度汇总报告文件"})`
5. **动作**：重跑 2021 年（先清理目录，再完整执行）

---

### 场景 B：合并 Excel 行数不对

**现象**：
```
year-2022/
  ├── 2022-report.json       # {"total": 600}
  └── 2022-merged.xlsx       # 实际只有 550 行
```

**校验过程**：
1. 目录有效性 → ✅
2. 无结果标记 → ❌
3. 文件存在 → ✅
4. 解析 `report.json` → `reported_total = 600`
5. 读取 `merged.xlsx` → `actual_rows = 550`
6. 计算差值：`|600 - 550| = 50 > 10`
7. 判断结果：`failures.append({"year":"2022", "reported_total":600, "actual_rows":550})`
8. **动作**：重跑 2022

**可能原因**：
- 导出过程中断，部分批次丢失
- 合并时去重导致行数减少（正常应在误差范围内）
- 下载的文件损坏

---

### 场景 C：文件夹为空（磁盘清理误删）

**现象**：
```
year-2023/    # 目录存在，但内部文件全被删除
```

**校验过程**：
1. `_is_year_output_valid` → `list(dir.iterdir()) == []` → `False`
2. 原因：`_get_year_output_invalid_reason` → "年份文件夹为空"
3. 判断结果：`failures.append({"reason": "年份文件夹为空"})`
4. **动作**：重跑 2023

---

### 场景 D：无结果（正常跳过）

**现象**：
```
year-2024/
  └── 2024-no-results.txt
```

**校验过程**：
1. 目录有效性 → ✅
2. `find_yearly_no_results_note` → 找到文件
3. `continue`（既不加入 failures，也不加入 skipped_year_details）
4. **结果**：该年标记为空，不参与合并，不计入失败

## 十、修复说明

### 问题定位

原 `vp-search` 的 `_collect_yearly_validation_outcomes` 方法存在逻辑错误：

```python
# 错误逻辑（已修复）
if report_path is None:
    skipped_year_details.append(...)   # ❌ 应该加入 failures
    continue
```

这会导致：
- 文件缺失的年份被标记为"跳过"
- 不触发重试机制
- 最终合并时该年数据丢失，但不报错

### 修复内容

1. **新增 `_is_year_output_valid` 检查**（与 cnki/wanfang 同步）
2. **统一 failures 收集逻辑**：
   - 目录无效 → failures
   - 报告文件缺失 → failures
   - Excel 文件缺失 → failures
   - 解析失败 → failures
   - 数据不一致 → failures
3. **保留 `skipped_year_details`** 仅用于：
   - 不在 `executed_years` 中的年份（未执行）
   - 在 `empty_years` 中的年份（无结果）

## 十二、故障排查速查

| 现象 | 可能原因 | 诊断 | 解决 |
|------|----------|------|------|
| 某年 `year-XXXX/` 目录不存在 | 该年检索未执行或执行失败 | 查 `executed_years` 列表 | 检查中断前的错误日志 |
| 目录存在但为空 | 导出中途清空或被删除 | 检查磁盘空间、手动删除记录 | 重跑该年（兜底会检测） |
| `report.json` 缺失 | 批次合并失败或未写入 | 查看该年目录下是否有 `batch-report.json` | 重跑该年 |
| `merged.xlsx` 行数不足 | 部分批次导出失败 | `python -c "import openpyxl; wb=openpyxl.load_workbook('...'); print(wb.active.max_row)"` | 重跑该年 |
| 行数差值 > 10 | 数据不一致（去重/重复） | 对比 `report.json` 的 total 与 Excel 行数 | 如非关键数据差异，可调大阈值 |
| 重试 2 次仍失败 | 网站反爬、网络问题、页面结构变化 | 查看是否频繁触发验证码 | 增加等待时间，检查选择器 |

## 十五、总结

vp-search 的模拟操作流程可归纳为：

1. **登录** → 保存会话（一次配置，长期复用）
2. **高级检索** → 填写表单 → 提交
3. **结果处理** → 逐页勾选 → 满 500 条导出一次
4. **逐年执行** → 循环执行第 2-3 步 N 个年份
5. **兜底校验** → 检查每个年份的输出完整性
6. **自动重试** → 对 failures 中的年份重跑（最多 2 轮）
7. **最终合并** → 汇总所有成功年份的 Excel
8. **生成报告** → 写入汇总信息

**关键特性**：
- 断点续传：意外中断可从进度文件恢复
- 容错性强：翻页失败自动重试、导出失败自动重试
- 数据校验：导出完成后自动对比报告与 Excel 行数
- 兜底机制：文件缺失或数据不一致时，自动清理并重跑
